Conceitos de Arquitetura

O que é Desenvolvimento Web

É o processo de criar e manter sites ou aplicações web. Envolve diversas etapas, como design, programação, teste, hospedagem e atualização
O desenvolvimento web pode ser dividido em duas áreas principais:
O front-end é a parte visível e interativa do site ou aplicação, que usa tecnologias como HTML, CSS e JavaScript
O back-end é a parte responsável eplo processamento de dados, lógica e segurança, que usa tecnologias como PHP, Python, Ruby, Java, entre outras.

Arquitetura Cliente-Servidor

  • A arquitetura cliente-servidor é um modelo de comunicação entre sistemas distribuídos, no qual um programa cliente solicita serviços ou recursos a um programa servidor.
  • O servidor processa a requisição e envia uma resposta ao cliente, que pode ser um arquivo, uma página web, um dado ou uma mensagen.

Protocolo HTTP

Protocolo

  • O Protocolo de Transferência de hipertexto, ou HTTP (Hypertext Transfer Protocol), é um protocolo de comunição utilizado para a transferência de dados na World Wide Web (WWW)
  • Ele define a forma como as solicitações e respostas devem ser formatadas e transmitidas entre um cliente (geralmente um navegador web) e um servidor web
  • O HTTP é a base para a troca de informações entre o usuário e os serviços web, permitindo que as páginas da web, imagens, vídeos e outros recursos sejam solicitados e exibidos em um navegador.

URLs

Uma URL (Uniform Resource Locators) é um identificador uniforme de recursos que especifica o endereço de um recurso na internet.

Uma URL é composta por vários componentes, como o protocolo, o domínio, o caminho e os parâmetros.

  • Por exemplo, a URL https://www.google.com/search?q=URL

tem os seguintes componentes:

  • Protocolo: https, que indica que a comunicação é segura e criptografada.
  • Domínio: www.bing.com, que identifica o servidor onde o recurso está hospedado.
  • Caminho: /search, que indica a localização do recurso dentro do servidor.
  • Parâmetros: q=URL, que especifica uma consulta de pesquisa para o recurso.

Uma URL permite que os usuários acessem e interajam com os recursos na internet de forma fácil e padronizada.

Métodos HTTP

Os métodos HTTP definem a ação que o cliente deseja executar no servidor.

Os métodos mais comuns são:

  • GET (obter dados), Esse método é usado para observa sem modificar nada.
  • POST (enviar dados para o servidor), Esse método cria algo novo no destino
  • PUT (atualizar dados), Esse método altera algo
  • DELETE (excluir dados) e outros, Esse método remove algo

Comportamento de uma Página no HTTP

Um documento completo é reconstruído a partir dos diferentes sub-documentos obtidos, como por exemplo texto, descrição do layout, imagens, vídeos, scripts e muito mais.

Clientes e servidores se comunicam trocando mensagens individuais (ao contrário de um fluxo de dados). As mensagens enviadas pelo cliente, geralmente um navegador da Web, são chamadas de solicitações (requests), ou também requisições, e as mensagens enviadas pelo servidor como resposta são chamadas de respostas (responses).

Request HTTP

Request (Requisição) é a mensagem enviada pelo cliente para o servidor solicitando algum recurso ou serviço. Normalmente as requisições vem acompanhadas:

  • Método HTTP: Indica a ação que o cliente deseja realizar (GET, POST, PUT ou DELETE)
  • URL: Identifica o recurso solicitado no servidor.
  • Headers: Contêm informações do Tipo de conteúdo (Content-Type), Informação de autenticação e Configurações de cache.
  • Body (corpo): Opcional, usado principalmente em métodos como POST ou PUT para enviar dados ao servidor
